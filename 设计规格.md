## 一、项目概况

纯前端像素风 Python 学习游戏。Vue 3 + Vite + TypeScript + Pyodide（浏览器内运行 Python）。
部署：GitHub Pages。存储：IndexedDB（idb 库）。

## 三、内容结构

### 3.1 阶段（Chapter）

| 阶段 | 名称 | 题量 | 状态 |
|------|------|------|------|
| 1 | 变量与类型 | 28 题（4 节 × 7 题） | ✅ 数据就绪 |
| 2 | 字符串与类型转换 | 28 题 | 🔄 待生成 |
| 3 | 条件判断 | 28 题 | 🔲 待生成 |
| 4 | 循环 | 28 题 | 🔲 待生成 |
| 5-10 | 各 16-22 题 | — | 🔲 待生成 |

### 3.2 节（Section）结构

每阶段拆为 4 节，每节 7 题。节间线性解锁。

```
第 1 章：变量与类型
├── 第 1 节：变量赋值（7 题）— 无前置
├── 第 2 节：类型系统（7 题）— 前置：第 1 节完成
├── 第 3 节：运算符与格式化（7 题）— 前置：第 2 节完成
└── 第 4 节：综合应用（7 题）— 前置：第 3 节完成
```

### 3.3 备用题库

每知识点（knowledgeTag）配 2 道备用题。每节最多插入 3 道备用题。
答错时从备用题库抽同知识点题目插入当前节末尾。

### 3.4 疲劳控制

| 条件 | 行为 |
|------|------|
| 单节答题数 ≥ 7 | 弹出休息提示（可跳过，默认停留 5s） |
| 连续答错 ≥ 3 | 诺克斯提示变为疲劳版本 |

## 四、数据模型

### 4.1 Question

```typescript
interface Question {
  id: string
  type: QuestionType
  chapterId: string
  title: string
  description: string
  difficulty: 1 | 2 | 3 | 4 | 5
  knowledgeTags: string[]
  stage?: string
  technicalDesc?: string
  narrativeTitle?: string
  narrativeDesc?: string
  narrativeExplanation?: string
  options?: string[]
  correctOption?: number
  initialCode?: string
  correctCode?: string
  expectedOutput?: string
  validationMode?: 'exact' | 'contains' | 'regex' | 'testcase'
  testCases?: TestCase[]
  commonErrors?: CommonErrorConfig[]
  explanation: string
  hint?: string
  hintRoleplay?: string
  hintDirect?: string
}
```

### 4.2 Chapter + Section

```typescript
interface ChapterSection {
  id: string
  name: string
  questionIds: string[]
  unlockAfter?: string
}

interface Chapter {
  id: string
  name: string
  description: string
  sections: ChapterSection[]
}
```

### 4.3 SectionProgress（Store）

```typescript
interface SectionProgress {
  completed: boolean
  questionResults: Record<string, { correct: boolean; score: number; attempts: number }>
  consecutiveWrong: number
}
```

## 八、架构图

```
App.vue (viewState)
  ├── loading → LoadingOverlay
  ├── chapterSelect → ChapterSelect (含节卡片展开)
  ├── wisdom → WisdomBook
  ├── quiz → QuizPanel (含疲劳控制)
  ├── chapterComplete → ChapterComplete

Data Flow:
  ChapterSelect → store.selectChapter() → WisdomBook → store.markWisdomViewed()
    → ChapterSelect 展开节卡片 → store.selectSection() → QuizPanel
      → store.submitSectionAnswer() → feedback → 补练/疲劳/下一题/完成
```

## 一、为什么学这些（不变）

列表、元组、字典、集合是 Python 的四种内置数据结构。

| 结构 | 为什么必须学 | 不学的后果 |
|------|------------|-----------|
| 列表 | 几乎所有 Python 程序都在处理列表 | 无法处理批量数据 |
| 元组 | 函数返回多值、数据保护、解包赋值的基础 | 无法理解 Python 多返回值 |
| 字典 | JSON、API、数据库记录——现代编程的数据交换格式 | 看不懂真实数据接口 |
| 集合 | 去重、交集/并集——算法题常见操作 | 用列表做集合操作 O(n²) |

**最终效果**：学习者能用 Python 表达"一组数据"，能读写 JSON 风格数据，能选择合适的数据结构。

## 七、s3_24 详细设计：卷宗编目系统（4★）

### 叙事场景

贤者在图书馆深处发现一批散落的典籍数据（模拟数据库查询结果）。需要编写一个函数 `catalog_books(data)`，按馆长的要求整理出可上架的目录。

### 输入数据格式

```python
data = [
    {"title": "元素基础", "shelves": 3, "floor": 1, "tags": ["火", "水"]},
    {"title": "中级炼金", "shelves": 5, "floor": 2, "tags": ["炼金", "实验室"]},
    {"title": "高级咒文", "shelves": 12, "floor": 3, "tags": ["咒文", "高阶"]},
    {"title": "草药图鉴", "shelves": 2, "floor": 1, "tags": ["采集", "森林"]},
]
```

### 任务要求

1. **过滤**：只保留 `shelves >= 3` 且 `floor <= 2` 的典籍
2. **转换**：返回一个字典，键为书名，值为所占书架数
3. **统计**：额外添加一个键 `"_total"`，值为所有符合条件典籍的 shelf 总和
4. **安全**：空列表 `[]` 返回 `{"_total": 0}`

### 考点覆盖

| 考点 | 对应操作 |
|------|---------|
| 列表遍历 | `for book in data` |
| 条件判断 | `book["shelves"] >= 3 and book["floor"] <= 2` |
| 字典写入 | `result[title] = shelves` |
| 字段读取 | `book.get("shelves")` 或 `book["shelves"]` |
| 边界处理 | 空输入、无条件匹配 |
| 累加 | `total += book["shelves"]` |

### 测试用例

| # | 输入 | 期望输出 |
|---|------|---------|
| 1 | `[{"title": "元素基础", "shelves": 3, "floor": 1}, {"title": "中级炼金", "shelves": 5, "floor": 2}, {"title": "高级咒文", "shelves": 12, "floor": 3}, {"title": "草药图鉴", "shelves": 2, "floor": 1}]` | `{"元素基础": 3, "中级炼金": 5, "_total": 8}` |
| 2 | `[]` | `{"_total": 0}` |
| 3 | `[{"title": "小册子", "shelves": 1, "floor": 1}, {"title": "笔记", "shelves": 2, "floor": 1}]` | `{"_total": 0}` |
| 4 | `[{"title": "元素基础", "shelves": 3, "floor": 1}, {"title": "高级咒文", "shelves": 12, "floor": 3}]` | `"高级咒文"` 不在结果中 |
| 5 | `[{"title": "边界测试", "shelves": 3, "floor": 2, "tags": ["测试"]}]` | `{"边界测试": 3, "_total": 3}` |

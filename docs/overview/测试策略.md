# 测试策略

> 明确我们测什么、怎么测、不测什么。

---

## 一、测试金字塔（由底向上）

```
         ╱ 人工走查 ╲          ← 验证叙事、体验、文案
        ╱ Playwright e2e ╲     ← 验证用户流程完整、判题正确
       ╱ 变体验证矩阵  ╲       ← 验证容错、边缘、一致
      ╱ vitest 单元测试 ╲      ← 验证纯函数（diff/errorParser）
     ╱  类型检查          ╲    ← 验证接口契约（vue-tsc）
```

### 1.1 类型检查（`vue-tsc --noEmit`）
- **目的**：接口违反即失败
- **范围**：所有 `.ts` / `.vue`
- **执行**：每次提交前

### 1.2 单元测试（vitest）
- **目的**：纯函数正确性（diff/errorParser/commonErrors）
- **现状**：57 个测试但被判定为虚假测试
- **计划**：ch3 扩展前重写，改为接触真实数据
- **不测**：组件渲染（不擅长 DOM 测试）

### 1.3 变体验证矩阵（Playwright）
- **目的**：验证判题系统对非标准输入的行为
- **范围**：每道 code_fill/fix/free_coding 题
- **覆盖**：空格变体、大小写、等价实现、常见错误
- **执行**：每批次新题执行一次
- **产出**：勘误表（哪些变体未通过预期）

### 1.4 E2E 流程验证（Playwright）
- **目的**：验证用户流程完整
- **场景**：
  - 场景 A：全对 → 节完成 → 解锁下一节
  - 场景 B：全错 → 备用题触发 → 上限 3 道
  - 场景 C：混合对错 → 重试模式 → 节完成
  - 场景 D：代码变体 → diff 显示
  - 场景 E/ch2：新章节全量验证
- **不测**：UI 像素、动画、视觉细节

### 1.5 人工走查
- **目的**：验证机器测不了的东西
- **覆盖**：叙事文案、游戏体验、引导流畅度
- **频率**：每次新增内容后

## 二、测试的核心矛盾

| ✅ 我们擅长测的 | ❌ 我们不擅长测的 |
|---------------|----------------|
| 管道通畅（提交→执行→反馈→下一题） | 内容质量（题目是否合理） |
| 判题一致性（同一输入→同一结果） | 学习效果（用户是否真的懂了） |
| 数据完整性（存档不丢） | 叙事体验（文案是否沉浸） |
| 流程完整性（解锁→答题→完成） | 视觉细节（像素排列、颜色） |

**结论：** 测试保证的是"机器不崩、判题一致"，不保证"内容好"。内容靠设计规范和人工走查。

---

## 三、判题验证哲学

判题只做三件事：

```
用户代码 → 执行 → 输出A
                   ↓
正确代码 → 执行 → 输出B → 比较 A 和 B → PASS/FAIL
```

我们不验证：
- 代码风格（那是 linter 的事）
- 性能（那是 benchmark 的事）
- 安全性（那是沙箱的事）

我们验证：
- A 和 B 的比较逻辑是否正确
- 用户看到的错误信息是否有用
- validationMode 是否按预期工作

---

## 四、测试不通过时怎么办

| 情况 | 处理方式 |
|------|---------|
| 正确答案被判 FAIL | **数据 bug**，修 expectedOutput / correctCode |
| 变体答案被判与预期不符 | **验证预期是否合理**，调整或修逻辑 |
| 类型检查不通过 | **必须修复**，不允许跳过 |
| e2e 流断裂（找不到按钮） | **组件 bug 或选择器失效**，优先排查 |
| e2e 超时 | Pyodide 加载慢 → 增加 timeout 或排查性能 |
| 节完成页面不出现 | 检查 `finishSection` 的触发条件 |

## 五、已知限制

| 限制 | 影响 | 补偿措施 |
|------|------|---------|
| `input()` 在 Pyodide 中无 stdin | s1_23/s1_25 不可自动验证 | 标记为"人工验证" |
| Pyodide 首次加载慢（5-10MB WASM） | e2e 超时风险 | timeout 设 90s |
| 无视觉对比测试 | UI 回归靠人工 | 在人工走查中覆盖 |
| 无性能基线 | 加载时间退化不可感知 | 后续加 LoadingOverlay 耗时记录 |

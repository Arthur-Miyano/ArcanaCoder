# ArcanaCoder 美术资产规范

> 本文档是给美术的唯一交付标准。所有素材必须按此规范命名、分类、存放。
> 风格：16-bit 像素风 | 主色调：深紫/深蓝 + 金/蓝强调色 | 硬边缘，无抗锯齿

---

## 一、通用规范

### 1.1 文件格式

| 类型 | 格式 | 原因 |
|---|---|---|
| 角色/道具/UI 精灵 | PNG-24（带透明通道）| 支持透明，无损 |
| 背景图 | PNG-24 或 JPG | PNG 优先，复杂背景可用 JPG |
| 动画序列 | PNG Sprite Sheet | 一图多帧，减少请求 |
| 可拉伸 UI | 九宫格切片 PNG | 适配不同尺寸 |

### 1.2 像素缩放规则

- **原图画小，运行时用 CSS 放大**
- 必须保留像素清晰度：前端使用 `image-rendering: pixelated`
- 例如：32×32 原图，游戏内显示 64×64（2x）或 96×96（3x）
- 禁止直接画大分辨率再缩小

### 1.3 颜色参考

| 用途 | 色值 |
|---|---|
| 背景主色 | `#1e1e32` |
| 卡片/石板背景 | `#1a1a2e` |
| 石板边框 | `#2d2d4a` |
| 强调金 | `#c9a227` |
| 强调蓝 | `#4a90e2` |
| 成功绿 | `#2ecc71` |
| 错误红 | `#e74c3c` |
| 文字主色 | `#e0e0e0` |

## 三、命名规范

### 3.1 通用规则

- 全部小写
- 用连字符 `-` 分隔单词
- 用下划线 `_` 分隔类别
- 动画标注帧数：`{name}-{frames}f.png`
- 状态变体用点分隔：`{name}.{state}.png`

### 3.2 示例

```
mage-idle-3f.png              # 贤者待机，3帧
mage-correct-3f.png           # 贤者答对，3帧
mage-portrait.png             # 贤者头像
bg-syntax-temple.png          # 语法圣殿背景
ui.slate.normal.png           # 普通奥术石板
ui.slate.gold.png             # 金色奥术石板
ui.button.normal.png          # 普通按钮
ui.button.hover.png           # 悬停按钮
icon-map.png                  # 地图导航图标
```

## 七、常见问题

### Q1：不会画 Sprite Sheet 怎么办？
可以每帧导出一张单独 PNG，命名如 `mage-idle-01.png`、`mage-idle-02.png`，前端工程师可以帮你拼成 Sprite Sheet。

### Q2：石板纹理能不能纯 CSS 实现？
可以。如果美术时间紧张，可以先不出石板纹理图，前端用 CSS 边框 + 内阴影模拟。但长期来看，纹理图会更有质感。

### Q3：背景图一定要 1024×768 吗？
是的，这是项目基准分辨率。如果以后要做 16:9 适配，可以提供 1920×1080 版本，命名加 `-hd` 后缀，如 `bg-syntax-temple-hd.png`。

### Q4：贤者装备会改变外观，需要多少图？
如果只做全身像展示，每个装备组合画一张全身像即可。如果要做精灵换装，需要拆分图层（长袍、法杖、头部），复杂度会高很多。建议第一阶段每个装备只出图标 + 一张全身展示图。

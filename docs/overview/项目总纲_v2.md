## 一、项目定位

**名称**：ArcanaCoder: 智慧圣殿编年史  
**类型**：像素风 Python 学习 RPG / 教育游戏  
**平台**：Web 浏览器（纯前端）  
**目标用户**：零基础编程小白  
**最终目标**：让学员成长为能独立完成项目、并通过中级 Python 岗位面试的技术员。

**核心体验一句话**：玩家扮演一位在魔法大陆寻求终极智慧的女性贤者，通过"施展魔法"（编写 Python 代码）解锁区域、提升位阶，而伙伴诺克斯（会说话的雪白猫头鹰）会陪伴、提示与总结。

**设计铁律**：
1. **先沉浸，后学习**：玩家首先感受到"我在魔法世界成长"，其次才是"我在学 Python"。
2. **诺克斯是朋友，不是老师**：提示不是宣读答案，而是"也踩过坑的朋友"分享经验。
3. **错误是剧情的一部分**：答错不是失败，而是"咒语波动不稳"。
4. **教学 > 答题**：每个知识点必须有智慧之书讲解 + 可实验沙盒 + 应用练习，不能只靠选择题和填空题。
5. **游戏化是骨架，不是皮肤**：世界地图、角色成长、伙伴进化、成就都是学习动机系统的一部分。

## 二、技术栈（确认不变）

| 层级 | 技术 | 说明 |
|---|---|---|
| 框架 | Vue 3 + Vite + TypeScript | 保持纯前端架构 |
| 状态管理 | Pinia | 用户进度、掌握度、成就 |
| 样式 | Tailwind CSS 3 + 自定义主题 | 深色魔法风格 |
| Python 运行时 | Pyodide（浏览器内） | 代码执行，无需后端 |
| 代码编辑器 | CodeMirror 6（vue-codemirror） | 答题、实验室、智慧之书 |
| 持久化 | IndexedDB（idb 库） | 本地保存进度 |
| 测试 | Vitest + Playwright | 单元测试 + E2E |
| 部署 | GitHub Pages + GitHub Actions | `.github/workflows/deploy.yml` |
| LLM 评审 | 用户自带 Key，前端直连 | 不引入后端，Key 不落地存储 |

**不引入**：Node.js 后端、vue-router、游戏引擎、浅色模式。

## 三、世界观与叙事结构

### 3.1 背景

在魔法大陆 **艾瑟瑞亚（Aetheria）**，魔法是可以被解析、重组与优化的"古老代码"。千年前的贤者将世界底层法则编织成 **"源语"** —— 有着严格语法、精确类型与强大逻辑的神秘文字。

玩家扮演年轻女性贤者，前往大陆尽头的 **智慧圣殿（Sanctum of Wisdom）**。道路被五大区域封锁，每个区域由一种"源语法则"守护。伙伴 **诺克斯（Nox）** 是导师遗物、源码之灵，额头有淡蓝色符文，会在困惑时歪着头用朋友语气提醒。

### 3.2 世界地图：五大区域 + 智慧圣殿

```

                    │
                    ▼
            [前端海岸] ──► [智慧圣殿]
```

## 六、新增机制详解

### 6.1 世界地图（WorldMap）⭐

- 用 SVG 绘制五大区域 + 智慧圣殿
- 节点状态：未点亮（灰色石头）、当前（蓝色呼吸水晶）、已通关（金色星星）
- 点击节点展开该区域章节/节卡片
- 通关章节后播放节点点亮动画 + 区域专属音效

### 6.2 咒语实验室（SpellLab）⭐

- 空白 CodeMirror 编辑器 + 运行按钮
- 输出区显示运行结果
- 诺克斯根据输出给出解释（预设规则 + 后期可选 LLM）
- 可保存"我的实验咒语"到本地
- 与贤者笔记打通

### 6.3 贤者笔记 / 法术书（Grimoire）⭐

- 自动记录所有学过的 WisdomPoint
- 每个知识点显示：掌握度、来源章节、相关实验数量
- 可快速跳转回顾智慧之书
- 游戏化：未点亮/半点亮/完全点亮的符文样式

### 6.4 贤者成长系统 ⭐

等级不再只是数字，而是解锁能力：

| 位阶 | 解锁 | 视觉变化 |
|---|---|---|
| Lv.1-3 | 基础试炼 | 深紫长袍 |
| Lv.4-6 | 解锁法术书 | 法杖微光 |
| Lv.7-9 | 解锁咒语实验室 | 长袍镶金边 |
| Lv.10+ | 解锁智慧圣殿入口 | 全身金色光晕 |

### 6.5 诺克斯进化 ⭐

诺克斯对话随玩家成长变化：

- **初期**："加油，你可以的！"（鼓励型）
- **中期**："你这次在字典上错了 3 次，我当年也这样。"（分析型）
- **后期**："这个设计选得不错，比我导师当年还简洁。"（平等型）

### 6.6 成就系统 ⭐

| 成就 | 条件 |
|---|---|
| 初识源语 | 完成第 1 章 |
| 无错大师 | 单节全对 |
| 坚持不懈 | 同一题重试 3 次后答对 |
| 实验狂人 | 在咒语实验室运行 10 次 |
| 诺克斯的朋友 | 查看过所有三级提示 |
| 法则收集者 | 法术书中点亮 50 个知识点 |
| 开源贤者 | 完成第一个 Capstone |

### 6.7 终局 Capstone：GitHub + LLM 评审 ⭐

#### 流程

```
解锁智慧圣殿
  → 阅读 Capstone 任务（魔法叙事包装）
  → 选择 LLM 提供商并输入 API Key
  → 阅读 Key 安全风险提示（强制勾选）
  → 开发本地项目并推送到 GitHub
  → 在 CapstonePanel 提交仓库链接
  → 系统拉取 README / 主要代码 / 测试文件
  → 调用 LLM 按 rubric 逐项评审
  → 诺克斯润色评审结果为朋友语气
  → 显示评分 + 改进建议
  → 通过后播放通关动画
  → 提示用户立即废弃 LLM API Key
```

#### Key 安全规范（强制）

- Key 只存 `sessionStorage`，页面刷新丢失
- 不写入 IndexedDB / localStorage / 服务器
- 请求直接发 LLM 官方 API
- 评审完成后清空内存中的 Key
- 显示"立即废弃 Key"弹窗，提供各平台 Key 管理链接

#### 两道 Capstone 题目

**Capstone 1：CLI 工具项目**
> 贤者，你已经掌握了源语的基础法则。现在请你为圣殿图书馆编写一个命令行管理系统，能够管理书籍的增删改查。要求使用 SQLite、argparse、类型注解、异常处理和 pytest 测试。

**Capstone 2：Web API 项目**
> 贤者，你的源语已经足够强大。请你为圣殿设计一套任务管理 API，使用 FastAPI 实现 CRUD、数据验证、日志记录，并编写测试。

### 6.8 答题界面设计规范 ⭐

**核心原则**：答题界面是学习停留时间最长的界面，必须保持简洁、聚焦、低干扰。

**材质选择**：答题界面的题目卡、编辑器容器、反馈面板使用 **奥术石板** 风格；其他界面（大厅、智慧之书、法术书、行囊等）保持羊皮卷风格不变。

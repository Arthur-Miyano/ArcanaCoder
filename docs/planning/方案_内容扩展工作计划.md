## 前置问题：Chapter 1 是否需要改动

### 已知冲突

| 问题 | 说明 |
|------|------|
| Ch1 s1_strings 与 Ch2 内容重叠 | Ch1 第 4 节已有 9 道字符串题（转义/sep/end/input/注释），与新 Ch2「字符串与输入输出」主题重叠 |

### 决策

1. **Ch1 不动** — s1_strings 覆盖的是「字符串作为基础类型」的使用（如何创建字符串、转义、print 参数），Ch2 覆盖的是「字符串作为可操作对象」的进阶（`.split()/.join()/.replace()`、编码、格式化变体）。两者定位不同，不重叠。
2. **现有 `ch2_lists` 占位数据删除** — 5 题 placeholder（q6-q10）作废，在新 Ch3 生成时重新出题。

### 最终 Ch1 状态

28 题 / 4 节 / 不变。前置依赖：无。

## 防回归清单

每次改动后执行：

```
npx vue-tsc --noEmit       # 0 error
npx vitest run             # 全部 passed
npm run build              # exit 0
```

以下已知模式避免重犯（来自 `findings.md`）：

| 模式 | 预防 |
|------|------|
| TDZ 死区 | `watch({immediate:true})` 必须在所有 ref 定义之后 |
| IndexedDB DataCloneError | 持久化前 `JSON.parse(JSON.stringify(state))` |
| GDD 术语裸字符串 | 新文案先查 GDD §5.1 禁用词表 |
| 结构改动后 UI 层没同步 | 加数据后人工走查用户路径一遍 |

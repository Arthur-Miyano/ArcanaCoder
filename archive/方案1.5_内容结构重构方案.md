# ArcanaCoder Phase 1.5 — 内容结构重构方案

> 涉及 4 项变更：变量名透明度 / 章→节拆分 / 备用题库 / 疲劳控制
> 前置方案：方案1.1 ~ 1.4（以下只列增量变更）

---

## 一、当前方案的问题（诊断）

| 问题 | 根因 | 影响 |
|------|------|------|
| initialCode 注释不明确 | stage1_questions.ts 中部分 code_fill 题写了"# 在这里编写"但没有说明期望的值 | 用户不知道应该填什么具体值 |
| 28 题线性排列 | 四个试炼(trial-1~3)只是标签，没有真正的解锁关系 | 用户可以直接跳到最后一道题，没有渐进感 |
| 没有备用题 | 每个知识点只有 1 道题 | 错题补练时没有新题可出，用户反复看同一道题 |
| 没有疲劳控制 | 可以连续做 28 题不休息 | 用户测试反馈“觉得累” |

---

## 二、变更说明（逐项）

### 2.1 变量名透明度

**改动范围：** 只改 `src/data/stage1_questions.ts`，不改代码逻辑。

**规则：**

```
code_fill / free_coding 题的 initialCode 必须包含：
1. 注释说明期望赋什么值
2. 如果值是固定的（如 "贤者"），明确写出
3. 如果值可以是任意值（如"你的名字"），注释写"（任意值）"
```

**示例（修复 s1_02）：**

```typescript
// 改前
initialCode: '# 将 "Hello, World!" 赋值给 message\n\n\n# 打印 message\n',

// 改后  
initialCode: '# 将字符串 "Hello, World!" 赋值给变量 message\n# 然后用 print() 打印 message\nmessage = ""\n',
```

**示例（修复 s1_25）：**

```typescript
// 改前
initialCode: '# 读取两个数字并求和\n\na = input()\nb = input()\n\n# 计算和\n',

// 改后
initialCode: '# 读取两个整数，计算它们的和\n# 用 int() 将 input() 的结果转为整数\n# 例如输入 3 和 5 应输出 8\na = int(input())\nb = int(input())\n',
```

### 2.2 章→节拆分

**当前结构（不改的）：**

```
ch1_variables → 28 题（平铺）
```

**新结构：**

```
ch1_variables
  ├── section_1_1 (变量赋值) — 7 题 — 解锁条件：无
  ├── section_1_2 (类型系统) — 7 题 — 解锁条件：section_1_1 通关
  ├── section_1_3 (运算符与格式化) — 7 题 — 解锁条件：section_1_2 通关
  └── section_1_4 (综合应用) — 7 题 — 解锁条件：section_1_3 通关
```

**数据层变更（`types/index.ts`）：**

```typescript
interface ChapterSection {
  id: string
  name: string
  questionIds: string[]
  unlockAfter?: string   // 前置节的 id，无则为空
}

interface Chapter {
  id: string
  name: string
  description: string
  sections: ChapterSection[]   // 取代 questionIds: string[]
}
```

**Store 层变更（`gameStore.ts`）：**

```typescript
interface SectionProgress {
  completed: boolean
  questionResults: Record<string, { correct: boolean; score: number; attempts: number }>
  consecutiveWrong: number      // 用于疲劳控制
}
```

新增动作：
- `selectSection(sectionId)` — 进入某个节
- `submitSectionAnswer(questionId, correct)` — 提交节内答题
- `isSectionUnlocked(sectionId)` — 检查前置节是否完成
- `getCurrentSection()` — 获取当前节信息

**UI 层变更：**

| 组件 | 改动 |
|------|------|
| `ChapterSelect.vue` | 点击章节后不再直接进答题，先进**节选择页** |
| 新增 `SectionSelect.vue` | 显示节列表，带解锁状态和进度 |
| `QuizPanel.vue` | 修改 questionIds 来源为当前节的 questionIds |
| `App.vue` | 新增 `viewState === 'sectionSelect'` |

### 2.3 备用题库

**数据层变更（`src/data/backup_questions.ts`）：**

每个知识点（knowledgeTag）对应 3-4 道备用题：

```typescript
export const backupQuestions: Record<string, Question[]> = {
  'print': [
    { /* 变体1：不同字符串 */ },
    { /* 变体2：多参数 */ },
    { /* 变体3：sep参数 */ },
  ],
  'f-string': [
    { /* 变体1：单变量 */ },
    { /* 变体2：多变量 */ },
    { /* 变体3：表达式 */ },
  ],
}
```

备用题只包含：id、type、chapterId、section、description、initialCode、correctCode、expectedOutput、knowledgeTags、explanation。**不包含** narrative 字段（节省内容工作量，后续有需要再加）。

**判题逻辑变更：**

当用户答错一道题且该题有备用题时：

```
1. 记录当前题答错
2. 不标记为"需要重试原题"，而是从备用题库中抽一道同知识点的新题
3. 新题插入到当前节末尾
4. 用户继续作答新题
```

**备用题生成策略：**不多写。先用 AI（你给的 prompt 模板）为阶段 1 的每个 knowledgeTag 生成 2 道备用题，累计约 60 道。阶段 2+ 后续补。

### 2.4 疲劳控制

在 `QuizPanel.vue` 中加入：

| 触发条件 | 行为 |
|---------|------|
| 当前节内连续答对/答错累计答题数 = 8 | 自动弹出休息提示（可跳过，默认停留 3s） |
| 当前节内连续答错 = 3 | 诺克斯提示变为疲劳版本 |
| 用户选择休息 | 退回节选择页，进度已保存 |

**Nox 疲劳提示：**

```
连续答错 3 次 → "贤者，你的魔力有些紊乱了。建议回到智慧之书复习一下。"
连续答题 8 题 → "你已经连续施法 8 次了，贤者。休息一下效果更好。"
```

## 四、实现顺序

```
P0: 变量名透明度修复（仅改数据，30min）
  1. 审查 stage1_questions.ts 中所有 code_fill / free_coding 的 initialCode
  2. 给每个注释加上明确的期望值说明

P1: 疲劳控制（仅改 QuizPanel + NoxDialog，1h）
  1. QuizPanel 加 consecutiveWrong 计数器
  2. 累计 3 次错误时弹出疲劳诺克斯提示
  3. 累计 8 题时弹出休息提示

P2: 备用题库（数据 + 判题逻辑，2h）
  1. 创建 backup_questions.ts（AI 生成阶段 1 的 60 道备用题）
  2. QuizPanel 判题逻辑加"答错→抽备用题→插入节末尾"

P3: 章→节拆分（架构级，3h）
  1. 类型定义：ChapterSection / SectionProgress
  2. questions.ts：ch1_variables 改为 sections
  3. gameStore：节进度 + 节解锁 + 迁移旧数据
  4. SectionSelect.vue：节列表 + 解锁状态
  5. App.vue + ChapterSelect：视图流转
  6. QuizPanel：题目来源改为当前节
  7. 测试：节解锁 / 节进度 / 节间切换
```

## 二、修复方案

### 2.1 `validationMode` 实现

**涉及文件：** `src/components/QuizPanel.vue`（`validateCode` 函数，第 99-119 行）

**当前代码：**
```javascript
if (q.expectedOutput !== undefined) {
  return { correct: output.trim() === q.expectedOutput, userOutput: output.trim() }
}
```

**修改后：**
```javascript
if (q.expectedOutput !== undefined) {
  const userOut = output.trim()
  const expected = q.expectedOutput.trim()
  if (q.validationMode === 'contains') {
    return { correct: userOut.includes(expected), userOutput: userOut }
  }
  return { correct: userOut === expected, userOutput: userOut }
}
```

**数据确认：** 全部数据中 `validationMode` 取值分布：
- `'exact'`（绝大多数）
- `'contains'`（stage1 有 3 题：s1_23, s1_24, s1_25；backup 有 4 题）

**注意：** s1_24 的 `type` 是 `output_predict`（选择题），不经过 `validateCode` 函数，所以它的 `validationMode: 'contains'` 是冗余字段。建议从中删除以保持数据准确。真正受 `contains` 影响的是 `code_fill` / `code_fix` / `free_coding` 类型。

无其他模式，所以只需要处理 `'contains'` 分支，其余走精确匹配。

### 2.2 所有题型组件加 `:key`

**涉及文件：** `src/components/QuizPanel.vue`（模板）

**原因：** `v-if`/`v-else-if` 在同类型组件切换时会复用实例，不加 `:key` 可能导致 userAnswer 残留、动画异常、CodeMirror 内容未清空。

**修改：** 三个组件全部加上 `:key="currentQuestion.id"`：

```html
<ChoiceQuestion
  :key="currentQuestion.id"
  v-if="currentQuestion.type === 'choice' || currentQuestion.type === 'output_predict'"
  ...
/>

<CodeQuestion
  :key="currentQuestion.id"
  v-else-if="currentQuestion.type === 'code_fill' || currentQuestion.type === 'code_fix' || currentQuestion.type === 'free_coding'"
  ...
/>

<OutputPredict
  :key="currentQuestion.id"
  v-else-if="currentQuestion.type === 'output_predict'"
  ...
/>
```

`v-if` 和 `v-else-if` 的 `:key` 可以相同，Vue 会正确处理。

### 2.3 副作用影响

| 修复 | 影响范围 | 回归风险 |
|------|---------|---------|
| validationMode 实现 | 所有 `contains` 模式的题改为子串匹配 | 低——exact 模式不受影响，contains 只会更宽容 |
| OutputPredict `:key` | QuizPanel 每次切换题目时强制重建组件 | 低——组件重建是 Vue 的标准模式 |

## 一、测试基础设施

### 1.1 现状

| 项目 | 当前状态 |
|------|---------|
| 测试框架 | 未安装 vitest / @vue/test-utils |
| 测试脚本 | package.json 无 test 命令 |
| 单元测试 | 0 个 |
| CI | GitHub Actions 只跑了 build，没有 test 步骤 |
| 已发生的回归 bug | free_coding 判错、弹窗不显示、通关条件错误——共 3 次 |

### 1.2 改动方案

**安装：**
```
npm install -D vitest @vue/test-utils jsdom
```

**测试脚本（package.json）：**
```json
"scripts": {
  "test": "vitest run",
  "test:watch": "vitest"
}
```

**测试覆盖范围：**

| 模块 | 测试内容 | 用例数 |
|------|---------|--------|
| `src/utils/diff.ts` | 5 种匹配类型（exact/case/whitespace/format/logic）、78.5 vs 78.50、中英文标点、标点归一化 | 8-10 |
| `src/utils/errorParser.ts` | 中文翻译（NameError→变量未定义等）、行号提取、诺克斯提示匹配 | 5-6 |
| `src/utils/commonErrors.ts` | 大小写检测、空白字符检测、str+int 检测、Levenshtein 变量拼写 | 6-8 |
| `src/stores/gameStore.ts` | submitAnswer 经验计算、升级、通关条件、retry 状态 | 5-6 |

**不测的：** UI 组件渲染（QuizPanel、WisdomBook 等）。组件渲染涉及 CodeMirror/Pyodide 依赖，模拟成本高于收益。这类变更通过 `npm run dev` 手动验证。

**CI 配置（.github/workflows/deploy.yml）：**

在 build 前加 test 步骤：

```yaml
- run: npm ci
- run: npm run test
- run: npm run build
```

### 1.3 实现步骤

```
Step 1: npm install -D vitest @vue/test-utils jsdom @vitest/coverage-v8
Step 2: 配置 vitest.config.ts（指向 src/，使用 jsdom，设覆盖率阈值）
Step 3: 写 diff.test.ts（12-15 个用例，覆盖边界：空串、大小写、浮点精度、中文标点、多行差异、纯空格）
Step 4: 写 errorParser.test.ts（5-6 个用例）
Step 5: 写 commonErrors.test.ts（6-8 个用例）
Step 6: 写 gameStore.test.ts——只测内存计算逻辑（经验/升级/通关），IndexedDB 持久化跳过或 mock
Step 7: 更新 CI workflow 加 test 步骤
Step 8: 确认 npm run test 全部通过
```

### 1.4 vitest.config.ts

```typescript
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    environment: 'jsdom',
    coverage: {
      provider: 'v8',
      reporter: ['text', 'json', 'html'],
      thresholds: {
        lines: 60,
        functions: 60,
        branches: 50,
      },
    },
  },
})
```

## 二、智慧之书版式重构

### 2.1 现状

当前 WisdomBook 是垂直单栏滚动布局。用户要求改为左右双栏书本式布局，含 3D 翻页动画。

### 2.2 改动方案

**涉及文件：** 只改 `src/components/WisdomBook.vue`。数据层不动。

**HTML 结构（替换当前的单栏滚动）：**

```html
<div class="book-scene">
  <button class="page-arrow left">←</button>

  <div class="book-container" :class="{ flipping }">
    <!-- 左页：目录/知识点列表 -->
    <div class="book-page left-page">
      <div class="page-content">
        <!-- 显示当前和相邻知识点 -->
        <div v-for="p in visibleLeftPages" :key="p.title">
          <h3>{{ p.title }}</h3>
        </div>
      </div>
    </div>

    <!-- 书脊 -->
    <div class="book-spine"></div>

    <!-- 右页：当前知识点完整内容 -->
    <div class="book-page right-page">
      <div class="page-content">
        <h2>{{ currentPoint.title }}</h2>
        <section>专业定义</section>
        <section>语法规则</section>
        <section>法阵演示 + 运行按钮</section>
        <section>设计哲学</section>
        <section>常见错误</section>
        <section>即时验证</section>
      </div>
    </div>
  </div>

  <button class="page-arrow right">→</button>
</div>
```

**CSS 3D 翻页：**

- `perspective: 1200px` 容器
- `.book-container` flex 容器，宽 900px，高 600px
- `.book-page` 各占 50%，渐变色背景模拟羊皮纸
- `.book-spine` 中间竖条，金色渐变
- 翻页时 `.right-page` 做 `rotateY(-180deg)` 动画，`transform-origin: left center`
- `transition: transform 0.6s cubic-bezier(...)`

**翻页逻辑：**

```typescript
function nextPage() {
  isFlipping.value = true
  setTimeout(() => {
    currentIndex.value++
    isFlipping.value = false
  }, 600)
}
```

**占位方案（无美术素材）：**

| 元素 | 占位 |
|------|------|
| 书页背景 | `linear-gradient(to right, #e8dcc8, #f4e4c1, #d4c4a0)` |
| 书脊 | `linear-gradient(90deg, #8B6914, #c9a227, #8B6914)` |
| 翻页箭头 | CSS 三角形 + 金色边框 |
| 页码 | 纯文字 "#1 / 5" |

**不动的：** 数据内容（definition、syntaxRules、code、designPhilosophy、commonErrors、interactiveCode 全保留）。

### 2.3 补充细节

**动画锁：** 翻页期间 `isFlipping` 加锁，快速点击不触发二次动画。

```typescript
function nextPage() {
  if (isFlipping.value) return
  isFlipping.value = true
  setTimeout(() => {
    currentIndex.value++
    isFlipping.value = false
  }, 600)
}
```

**左页目录：** 不显示"相邻知识点"，改为完整知识点目录（类似书的目录页），当前知识点高亮。

**移动端回退：** 768px 以下改为垂直单栏，取消 3D 变换，隐藏书脊。

```css
@media (max-width: 768px) {
  .book-container {
    flex-direction: column;
    width: 100%;
    height: auto;
    perspective: none;
  }
  .book-page { width: 100%; transform: none !important; }
  .book-spine { display: none; }
}
```

**CodeMirror 保护：** 翻页动画期间 CodeMirror 容器 `opacity: 0`，动画完成后恢复。避免 3D 变换导致光标坐标偏移。

### 2.4 实现步骤

```
Step 1: 重写 WisdomBook.vue 的 <template> 为双栏书本结构
Step 2: 添加 <style scoped> 中的 3D CSS
Step 3: 添加翻页交互逻辑（nextPage/prevPage）
Step 4: 右页内容保持不变（映射到 currentPoint 的各字段）
Step 5: 左页显示知识点目录列表
Step 6: 适配翻页箭头位置
Step 7: 验证：npm run dev 手动测试翻页效果
```

## 三、总结：需要你确认的

| 事项 | 做不做 | 优先级 |
|------|--------|--------|
| 安装 vitest 写核心模块测试 + CI 集成 | 建议做 | 高——防止再次出现回归 bug |
| 智慧之书双栏 3D 翻页重构 | 建议做 | 中——纯 UI 改动，不影响功能 |
| 测试先后顺序 | 先写 diff / errorParser / commonErrors 测试（纯函数，不依赖 Vue）→ gameStore → 最后 UI |

同意的话我按 测试 Step 1 → 测试 Step 2 → …… → 智慧之书 Step 1 的顺序实施。

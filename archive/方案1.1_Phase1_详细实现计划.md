> 核心目标：能打开网页、能做题、能运行 Python、有即时反馈

## 一、技术选型确认

| 层面 | 选型 | 理由 |
|------|------|------|
| 构建 | Vite 6 + Vue 3.5 + TypeScript 5.7 | 路线图指定 |
| 状态管理 | Pinia | 路线图指定 |

## 四、数据模型

### 4.1 题目类型（5 种）

```typescript
type QuestionType = 
  | 'choice'          // 单选题
  | 'code_fill'       // 代码填空
  | 'code_fix'        // 代码纠错
  | 'output_predict'  // 预测输出
  | 'free_coding'     // 自由编程
```

### 4.2 Question 类型定义

```typescript
interface Question {
  id: string
  type: QuestionType
  chapterId: string
  title: string
  description: string       // 题干
  difficulty: 1 | 2 | 3
  knowledgeTags: string[]   // 知识点标签（Phase 2 掌握度用）

  // choice / output_predict
  options?: string[]
  correctOption?: number    // 正确答案索引

  // code_fill / code_fix / free_coding
  initialCode?: string      // 初始代码（编辑器内容）
  correctCode?: string      // 参考正确答案

  // output_predict
  expectedOutput?: string   // 预期输出（用于自动比较）

  // 通用
  explanation: string       // 解析
  hint?: string
}
```

### 4.3 AnswerRecord 类型

```typescript
interface AnswerRecord {
  questionId: string
  correct: boolean
  answer: string           // 用户答案（JSON 序列化）
  score: number            // 0-100
  timestamp: number
  attemptCount: number
}
```

### 4.4 Chapter 类型

```typescript
interface Chapter {
  id: string
  name: string
  description: string
  questions: string[]       // Question.id 列表
  requiredLevel?: number    // 解锁所需等级
}
```

## 五、5 道测试题设计

一个关卡"第1关：变量与类型"，5 道题覆盖全部 5 种题型。

| # | 题型 | 题目 | 知识点 |
|---|------|------|--------|
| 1 | choice | Python 中哪个是字符串类型的值 | 字符串字面量 |
| 2 | code_fill | 补全代码：用变量 `name` 存储 "世界"，然后打印 `你好，世界` | 变量赋值 + f-string |
| 3 | code_fix | 修复代码中的错误（缩进/类型错误） | 缩进规则 |
| 4 | output_predict | 阅读代码，预测输出结果 | print + 类型转换 |
| 5 | free_coding | 编写一个函数计算圆的面积 | 函数定义 + 数学运算 |

**每题的具体内容见 `src/data/questions.ts`。**

## 六、Pinia 游戏状态 (gameStore)

```typescript
interface GameState {
  // 基础属性
  level: number               // 当前等级（初始 1）
  exp: number                 // 当前经验值
  expToNext: number           // 升级所需经验（公式：100 * level）

  // 进度
  unlockedChapters: string[]  // 已解锁关卡 ID
  currentChapterId: string | null

  // 答题记录
  chapterProgress: Record<string, {
    completed: boolean
    questionResults: Record<string, {
      correct: boolean
      score: number
      attempts: number
    }>
  }>

  // 元数据
  lastPlayedAt: number | null
  totalQuestionsAnswered: number
  totalCorrect: number
}
```

**Actions：**
- `submitAnswer(questionId, answer)` → 判题 → 更新 exp/level → 存储到 IndexedDB
- `selectChapter(chapterId)` → 设置当前关卡
- `loadFromStorage()` → 从 IndexedDB 恢复状态
- `resetProgress()` → 重置所有进度

**升级公式：**
- 升级所需经验 = `100 * level`
- 每题得分：choice 10exp, code_fill 20exp, code_fix 20exp, output_predict 15exp, free_coding 30exp
- 答对全额给，答错给 1/3 保底经验
- 升级时 level++，exp 重置为超出部分

## 七、服务层设计

### 7.1 Pyodide 服务 (`pyodide.ts`)

```typescript
// 单例模式，全局只加载一次
async function initPyodide(onProgress?: (progress: number) => void): Promise<Pyodide>
async function runPython(code: string): Promise<{ output: string; error: string | null }>
function isReady(): boolean
function getPyodide(): Pyodide | null
```

**加载策略：**
1. 从 CDN 加载 `pyodide.js`（约 200KB）
2. 自动加载 `micropip` 和常用包
3. 通过 Pyodide 的 `registerModule` 捕获 `print` 输出
4. 加载期间通过回调上报进度（0-100）

**代码执行安全：**
- 设置 `timeout` 机制（默认 10 秒）
- 捕获 Python 异常并显示

## 八、组件详细规约

### 8.1 LoadingOverlay.vue
**Props:** `progress: number` (0-100)
**显示内容：**

- 进度条（Tailwind bg-purple-500）
- 进度文本："贤者正在准备魔法... 66%"
- 加载完成后自动切换到下一视图

### 8.2 GameHeader.vue
**Props:** `level: number`, `exp: number`, `expToNext: number`
**显示内容：**
- 紫色圆块（32x32）→ 贤者头像占位
- "贤者 Lv.{level}"
- 经验条：`████░░░░░░ 50/100`
- 经验百分比文字

### 8.3 ChapterSelect.vue
**Props:** 通过 store 获取数据
**显示内容：**
- GameHeader 置顶
- 关卡列表，每项一个卡片：
  - 关卡名称（如"第1关：变量与类型"）
  - 简短描述
  - 灰色方块占位（16x16）
  - 完成状态（✓ / 未完成）
- 点击卡片 → 切换到 quiz 视图

### 8.4 QuizPanel.vue
**Props:** `chapterId: string`
**逻辑：**
- 根据 store 获取当前章节题目列表
- 显示当前题号（第 2/5 题）
- 根据题目类型动态渲染子组件
- 提交答案后调用 store.submitAnswer()
- 显示 FeedbackToast
- 所有题完成后显示章节完成页

### 8.5 ChoiceQuestion.vue
**Props:** `question: Question`, `modelValue: number | null`
**逻辑：**
- 渲染单选按钮列表（○/● 样式）
- 选中状态用紫色高亮
- 点击选项后自动触发父组件提交（或由父组件控制提交按钮）

### 8.6 CodeQuestion.vue
**Props:** `question: Question`, `modelValue: string`
**逻辑：**
- 集成 CodeMirror 6（vue-codemirror）
- language: python
- 初始内容来自 question.initialCode
- 用户可编辑
- 提交时：如果是 code_fix/free_coding，将代码发送到 Pyodide 运行并比较输出

### 8.7 OutputPredict.vue
**Props:** `question: Question`
**逻辑：**
- 显示一段代码片段（只读 CodeMirror）
- 下方显示多个输出选项（选择题形式）
- 提交时：可选"运行验证"按钮 → 用 Pyodide 运行代码显示实际输出
- 判题：选择与预期输出一致即正确

### 8.8 FeedbackToast.vue
**Props:** `visible: boolean`, `correct: boolean`, `message: string`, `explanation: string`
**逻辑：**
- 答对：绿色背景 + 贤者答对表情（纯色块）
- 答错：红色背景 + 贤者答错表情
- 显示解析文本
- 3 秒后自动消失或点击"下一题"关闭
- Phase 1: 纯色块 + 文字

### 8.9 NoxDialog.vue
**Props:** `message: string`
**逻辑：**
- 白色圆块（16x16）→ 诺克斯头像占位
- 右侧文字气泡
- 用于：答题前的提示、完成后的鼓励、加载中的等待语

## 九、数据流

```
用户操作 → 子组件事件 → QuizPanel → gameStore.submitAnswer()
                                            ↓
                                  判题逻辑（代码题→Pyodide）
                                            ↓
                             更新 exp/level/completedQuestions
                                            ↓
                                   FeedbackToast 显示
                                            ↓
                                  storage.saveGameState()
                                            ↓
                                  自动或手动切下一题
```

**代码题判题流程：**
```
提交代码 → Pyodide.runPython(code)
         → 获取 stdout + stderr
         → code_fill: 比较正确代码的输出（忽略空格）
         → code_fix: 比较修复后代码的输出是否符合预期
         → free_coding: 运行测试用例（简单: 检查输出是否包含关键词）
         → 返回 { correct, output, error, score }
```

## 十一、风险点 & 决策

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| Pyodide 10MB+ WASM 加载慢 | 用户等待时间长 | LoadingOverlay 进度条 + 缓存（IndexedDB 可缓存 WASM） |
| CodeMirror 6 包体积 | 增加 JS bundle | 按需加载，仅 code 题型才 import |
| 代码题判题准确率 | 用户困惑 | Phase 1 用宽松比较（输出包含关键词即对，忽略首尾空格） |
| IndexedDB 异步 | 状态管理复杂度 | 封装在 storage.ts 中，store 同步操作 storage 异步落盘 |
| GitHub Pages SPA 路由 | 刷新 404 | 用 hash 模式或 404.html 重定向（但 Phase 1 无路由，不涉及） |

## 四、为什么之前没发现这些问题

| 根因 | 说明 |
|------|------|
| **验证清单缺 GDD 审查** | 设计规格 §9.2 的人工走查路径只检查功能不崩溃，不检查文案是否符合世界观 |
| **AI 生成题目时没有 GDD 约束 prompt** | 生成 stage1_questions.ts 时的 prompt 没有要求 narrativeExplanation |
| **测试只覆盖逻辑和编译** | 37 个 vitest 测试没有一个检查 UI 文案 |
| **关注点偏技术实现** | 修复 feedback、diff、retry 时只关注功能正确性，没回头检查文案 |

## 五、优化建议

### 5.1 GDD 审查加到验证清单

在 `设计规格.md` §9.2 增加一条人工走查：
> 新增文案（按钮、提示、反馈）→ 对照 GDD §5.1 术语替换表逐字检查

### 5.2 术语表用常量管理

```typescript
// src/utils/vocabulary.ts
export const GDD_TERMS = {
  correct: '魔力共鸣',
  incorrect: '咒语波动不稳',  
  submit: '注入魔力',
  question: '试炼',
  // ...
}
```

组件引用常量而非硬编码文案。

### 5.3 narrativeExplanation 补全

28 道题需要补充 narrativeExplanation。采用批量生成（AI prompt 模板 + 人工审校）。

### 5.4 Knox 提示 tone 升级

约 15 道题的 hintRoleplay 需要从"中性提示"升级为"朋友口吻"，加"嘿""哈哈""我当年..."等开放式表达。

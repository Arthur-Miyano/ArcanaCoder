## 第二章：核心变更 — 题目生成方式

### 2.1 放弃自动导入脚本

30-Days-Of-Python 是教程文本不是题库，且 Markdown 格式不统一。
导入脚本的投入产出比极低（写 2h 脚本 + 更多时间手工修）。

### 2.2 改用 AI Agent 批量生成题目

**流程：**

```
参考素材（HackerRank / LeetCode / 任何 Python 题目）
    ↓
AI Prompt（你给的模板）
    ↓
输出 ArcanaCoder 格式 JSON（choice / output_predict / code_fill）
    ↓
人工审查（调整难度、补叙事描述）
    ↓
写入 src/data/questions.ts
```

**一个素材生成 3 道题**（choice + output_predict + code_fill），
每个素材的题量自然膨胀到 3 倍。

### 2.3 题目素材来源

| 优先级 | 来源 | 题量 | 适合阶段 |
|--------|------|------|---------|
| P0 | HackerRank Python 题库 | 约 40+ 题 | 阶段 1-4 |
| P0 | LeetCode 简单/中等 | 约 100+ 题 | 阶段 3-7 |
| P1 | Codewars Python 题 | 大量 | 阶段 4-10 |
| P2 | 自己编写（覆盖 v3.0 知识标签） | 按需 | 全部 |

### 2.4 生成示例（基于你的 Prompt）

你的 Prompt 模板已经可以直接用了。我会：

1. 取一个 HackerRank 题目原文
2. 塞进你给的 Prompt 模板
3. 生成 3 道题（choice/output_predict/code_fill）
4. 输出 JSON 验证
5. 批量重复

需要你确认的是：**素材优先从哪个来源取？** HackerRank / LeetCode 还是别的？

## 第四章：本地版 Capstone

### 4.1 不做 GitHub 提交，做本地验证

Capstone 在 Phase 1 的形态：

```
用户看到任务书（题目描述）
  → 在 CodeMirror 中写完整代码
    → 点击 [提交项目]
      → Pyodide 运行多组测试用例
        → 全部通过 → 显示"阶段完成" + 额外经验奖励
        → 有失败 → 显示测试报告（哪个 case 没过，期望/实际是什么）
```

### 4.2 需要新增的

| 组件 | 说明 |
|------|------|
| CapstoneTask.vue | 任务书展示 + 代码编辑器 + 提交按钮 |
| 测试报告显示 | FeedbackPanel 的增强版，多 case 结果列表 |

**不改现有 QuizPanel。** Capstone 是独立视图，在 App.vue 里新增 `viewState === 'capstone'`。

## 第五章：commonErrors 模式匹配

### 5.1 匹配引擎

```typescript
interface CommonError {
  name: string                    // 错误代号，如 "CASE_MISMATCH"
  detect: (userOutput: string, expected: string) => boolean
  message: string                // 技术说明
  noxHint: string                // 诺克斯提示
}
```

### 5.2 预置检测器（Phase 1 实现）

| 检测器 | 检测逻辑 | 诺克斯提示示例 |
|--------|---------|---------------|
| 大小写不匹配 | `a.lower() === b.lower()` | "你写的字符大小写和期望的不完全一致。Python 是大小写敏感的，'hello' 和 'Hello' 是两个不同的字符串。" |
| 空白字符差异 | `a.trim() === b.trim()` | "你的输出多了一个空格或少了一个换行。print() 默认会在末尾加换行，可以用 end='' 控制。" |
| Levenshtein 近似变量 | 用户代码中的变量名与期望变量名的编辑距离 ≤ 2 | "你是不是把 area 写成了 arer？就差一个字母。" |
| str + int 模式 | 检测 `"string" + number` 模式 | "字符串和数字不能直接加。用 str(数字) 把数字转成字符串，或者用 f-string。" |

### 5.3 匹配结果展示

匹配到的 commonErrors 显示在 FeedbackPanel 的"差异分析"区域，优先级高于通用错误信息。
